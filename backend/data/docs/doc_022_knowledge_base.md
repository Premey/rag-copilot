# Knowledge Base Setup

The CloudDesk Knowledge Base lets you publish self-service help articles so customers can solve issues without filing tickets.

## Creating Articles
1. Knowledge Base → New Article.
2. Enter title, category, and body (rich text editor with Markdown support).
3. Add meta description for SEO.
4. Attach images or files if needed.
5. Set visibility: **Public** (anyone can view), **Internal** (agents only), or **Draft** (not published).
6. Click Publish.

## Categories & Structure
Organise articles into categories and subcategories. Settings → Knowledge Base → Categories → New Category. Drag to reorder.

## Article Drafts & Versioning
All saves create a draft version. Published articles maintain version history — click 'History' to view or restore past versions.

## Linking to Tickets
From any ticket, click 'Suggest Article' to search the knowledge base and insert a link into your reply.

## SEO & Public Portal
Your public knowledge base is hosted at `yourcompany.clouddesk.io/help`. Custom domains available on Pro and Enterprise. Google-indexable by default.

## Analytics
Knowledge Base → Analytics: views per article, search queries with no results (use these to identify gaps), deflection rate (tickets avoided because a customer found a KB article).
